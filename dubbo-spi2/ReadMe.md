# dubbo spi

## 简单概念
```aidl
拓展点：接口 @SPI 
拓展：接口实现类 @Adaptive
拓展自适应：根据服务端配置或客户端参数，决定调用服务实现。

1. @SPI("xxx") 注解在接口上，声明接口的默认实现
2. @Adaptive 注解在接口实现类上，声明接口默认实现
3. @Adaptive 注解在接口抽象方法上，url上以 adaptive.ext2=xxx 参数形式，由客户端选择实现方式
4. @Adaptive({"key"}) 注解在接口抽象方法上，url上以 key=xxx 参数形式，由客户端选择实现方式

优先级：2 > 3,4 > 1

```

## 优点
```aidl
无侵入性、无修改、基于配置 拓展接口 (实现方式 与接口定义，解耦拓展，客户端可自主选择实现类)
```
